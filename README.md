# Couples Resort: Final Integrated Project

This repo is for the final integrated project for semester one of Interactive Media Design at Fanshawe College

This project builds upon the Couples Resort website developed during the first semesters classes, expanding it with three new mobile pages (412px) and three equivalent desktop pages (1200px). All pages are designed, wireframed, and prototyped in Figma, then developed using static CSS Grid layouts with validated HTML.

The design follows the established Couples Resort branding, focusing on consistency, alignment, contrast, typography, colour, UX/UI, and visual balance. Each page demonstrates principles of responsive design and accessibility while maintaining a cohesive brand experience.

## Features & Requirements:

✅ Static CSS Grid layouts for mobile and desktop
✅ CSS animation for a submit button
✅ ::after pseudo-element for a completion indicator
✅ Show/hide functionality (hamburger menu or details/summary)
✅ Proper Git workflow (branches, pull requests, merges)
✅ Includes wireframes, high-fidelity prototypes, and supporting assets

## Installation: 

1. Download or clone the project repository from GitHub.
2. Open the project folder on your computer.
3. If you are using Sass (SCSS), make sure you have Sass installed on your system. You can install it globally using the command: npm install -g sass, or if you’re working in VS Code, you can install the live sass compiler extension.
4. Compile the SCSS files into CSS before running the project. You can use the Sass watch command to automatically compile changes as you edit your styles.
5. Once the SCSS files are compiled, all styles will appear in the CSS folder and be linked to the HTML files.

---

## Usage

1. Open the index.html file in your web browser to view the project.
2. You can also use a local development server such as Live Server to preview your site and test interactive features.
3. Navigate through the available pages to explore the design. 
4. Test the interactive features, including:

    - A CSS animation applied to the submit button (reservations page).
    - A completion indicator created using the ::after pseudo-element (contact forms on the home / activities pages).
    - Show and hide functionality for the navigation (hamburger menu and details/summary element).

5. To make updates, edit the HTML, SCSS, or JavaScript files as needed. The SCSS files will need to be recompiled into CSS after changes are made.
6. Follow standard workflow practices when updating the project. Create a branch, make your changes, commit them, and then merge them into the main branch.
